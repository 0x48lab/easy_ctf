Admin Guide

EasyCTF setup and operation methods

Setup

Requirements

  • Minecraft Paper Server 1.21.5 or higher
  • Java 21
  • Minimum 2GB RAM recommended

Installation Steps

  1. Download

    Download the latest jar file from Releases

  2. Placement

    Place in the server's plugins folder

  3. Startup

    Start the server and the plugins/EasyCTF/ folder will be generated

  4. Configuration

    Edit config.yml to adjust basic settings

Creating Game Arenas

Interactive Creation

/ctf create arena1
1

Red Team Flag Position

Look at where you want to place the flag and type "set"

Beacon and iron block base are automatically placed

2

Red Team Spawn Point

Look at where you want the spawn point and type "set"

Must be at least 3 blocks away from the flag

3

Blue Team Setup

Set flag position and spawn point for blue team same as red team

4

Phase Settings

Select build phase game mode and set duration for each phase

Arena Design Points

Recommended Size

  • Minimum: 50×50 blocks
  • Recommended: 100×100 blocks
  • Maximum: 200×200 blocks

Layout

[Red Spawn]---[Red Flag]
     |           |
  (Central Area)
     |           |
[Blue Flag]---[Blue Spawn]

Considerations

  • Symmetry between both teams
  • Multiple invasion routes
  • Utilization of elevation differences
  • Natural obstacle placement

Configuration File Details

config.yml

Basic Settings

# Language setting
language: "en"  # "en" or "ja"

# Default game settings
default-game:
  min-players: 2            # Minimum number of players
  max-players-per-team: 10  # Maximum players per team
  respawn-delay: 5          # Respawn wait time (seconds)
  force-pvp: true           # Force PvP enable

Phase Settings

# Phase settings
default-phases:
  build-duration: 120       # Build phase (seconds)
  build-phase-gamemode: "ADVENTURE"  # ADVENTURE/SURVIVAL/CREATIVE
  combat-duration: 120      # Combat phase (seconds)
  result-duration: 60       # Result phase (seconds)

Currency Settings

# Currency settings
currency:
  name: "G"
  initial: 50               # Initial currency
  phase-end-bonus: 50       # Phase end bonus
  kill-reward: 10           # Kill reward
  carrier-kill-reward: 20   # Flag carrier kill reward
  capture-reward: 30        # Capture reward

Permission Management

Permission Description Default
ctf.use Basic command usage Everyone
ctf.admin Admin command usage OP
ctf.* All permissions OP

LuckPerms Configuration Example

groups:
  player:
    permissions:
    - ctf.use
  
  moderator:
    permissions:
    - ctf.use
    - ctf.admin.stop
    
  admin:
    permissions:
    - ctf.*

Operation Tips

Performance Optimization

  • Recommended to run max 3-4 games simultaneously
  • Large matches (20 players) should be run one at a time
  • Regular server restarts recommended

Troubleshooting

Game won't start

  • Check if flags and spawn points are set
  • Check if minimum player count is met

Lag occurs

  • Check entity count
  • Reduce number of concurrent games
  • Increase server RAM

Backup

Files to backup regularly:

  • plugins/EasyCTF/config.yml
  • plugins/EasyCTF/games/ folder
  • plugins/EasyCTF/lang_*.yml

Event Hosting Tips

Tournament Format

  1. Prepare multiple arenas
  2. Create tournament bracket
  3. Each match is first-to-3 wins
  4. Finals are first-to-5 wins

Tournament Rules Example

  • Team size: Fixed 5v5
  • Build phase: 3 minutes
  • Combat phase: 7 minutes
  • Banned items: TNT
  • Special rules: Ender pearl usage restrictions

Streaming/Spectator Settings

  • Prepare elevated areas for spectators
  • Set game mode to spectator
  • Dedicated permission group for streamers

Update Procedure

  1. Stop the server
  2. Backup configuration files
  3. Replace with new jar file
  4. Start the server
  5. Check configuration file compatibility
Note: Major updates may require configuration file changes.
