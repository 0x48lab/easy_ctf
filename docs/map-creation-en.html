Map Creation Guide

Create balanced and enjoyable CTF maps

üó∫Ô∏è What is Map Creation

In EasyCTF, you can create and save dedicated CTF maps in advance. Created maps are automatically saved and restored to temporary worlds when games start.

Benefits: Using pre-created maps enables balanced and fair games. You can also create multiple different maps to provide various gaming experiences.

üöÄ Map Creation Process

1

Set Map Region

First, specify the range of the area to be used as a map.

/ctf setpos1 <game-name>

Execute while standing at the starting position

/ctf setpos2 <game-name>

Execute while standing at the ending position

By specifying two diagonal points, a rectangular region is set
2

Place Required Blocks

Place the following special blocks within the set region.

Block Purpose Quantity
Red Concrete Red team spawn point Only 1
Blue Concrete Blue team spawn point Only 1
Beacon + Red Glass Red team flag position 1 set
Beacon + Blue Glass Blue team flag position 1 set
Important: Flags and spawn points must be at least 3 blocks apart
3

Save the Map

Once block placement is complete, save the map.

/ctf savemap <game-name>

The system automatically detects blocks and creates game settings.

Saved maps are stored in compressed format at plugins/EasyCTF/maps/<game-name>.yml

üé® Map Design Tips

Symmetry

Aim for symmetrical map design to ensure fairness for both teams.

Multiple Routes

Provide multiple attack routes to increase tactical options.

Elevation

Three-dimensional structures enable strategic gameplay.

Defensive Structures

Place moderate defensive structures around flags to balance offense and defense.

Line of Sight

Use walls and obstacles to limit visibility, enabling surprise attacks and stealth.

Central Area

A strategically valuable central area can create intense battles.

‚öôÔ∏è Temporary World System

Saved maps are automatically restored to temporary worlds when games start.

Independent Worlds

A dedicated world (ctf_temp_<game-name>) is created for each game.

Protect Original World

Destruction and changes during games don't affect the original world.

Auto Cleanup

Temporary worlds are automatically deleted when games end.

Multiple Game Support

Multiple games can run simultaneously on different maps.

‚ùå Common Errors

Required blocks not found

Check that required blocks (concrete, beacon+glass) are placed within the specified region.

Multiple blocks of the same type found

Only one concrete block per color and one beacon per team can be placed. Remove extra blocks.

Flag and spawn too close

The distance between flag and spawn point is less than 3 blocks. Adjust placement.

Map region not set

Set the map region using /ctf setpos1 and /ctf setpos2.

üìã Map Creation Checklist

Pre-Creation Preparation

  • Work in creative mode
  • Secure sufficient flat area
  • Prepare necessary blocks

Required Element Placement

  • Set map region (setpos1/setpos2)
  • Placed one red concrete
  • Placed one blue concrete
  • Placed beacon + red glass
  • Placed beacon + blue glass
  • Flags and spawns are 3+ blocks apart

Design Verification

  • Map is symmetrical and well-balanced
  • Multiple attack/defense routes exist
  • Appropriate defensive structures present
  • Utilizes elevation differences

Final Check

  • Saved map with savemap command
  • No error messages displayed
  • Test play showed no issues
