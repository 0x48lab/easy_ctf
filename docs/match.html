マッチモード

複数ゲームを連続実施して総合優勝を決定

概要

マッチモードでは、複数のゲームを連続して実施し、総合的な勝者を決定します。単発のゲームよりも戦略性が高く、逆転のチャンスもある白熱した戦いが楽しめます。

マッチの種類

先取モード（first_to_x）

指定した勝利数に最初に到達したチームが優勝します。

例: 3勝先取の場合

  • 先に3勝したチームがマッチ優勝
  • 最短3ゲーム、最長5ゲームで決着
  • 緊張感のある展開

固定ラウンドモード（fixed_rounds）

指定した回数のゲームを行い、最多勝利チームが優勝します。

例: 5ラウンド固定の場合

  • 必ず5ゲーム実施
  • 最終的に勝利数が多いチームが優勝
  • 同数の場合は引き分け

マッチの開始方法

コマンド形式

/ctf start <ゲーム名> <モード> <目標値>

使用例

3勝先取マッチ

/ctf start arena1 first_to_x 3

最初に3勝したチームが優勝

5ラウンド固定マッチ

/ctf start arena1 fixed_rounds 5

5ゲーム実施して勝利数で決定

マッチ中の仕様

ゲーム間のインターバル

  • 各ゲーム終了後、5秒間のインターバル
  • 勝敗結果と現在のスコアを表示
  • BossBarでカウントダウン表示
  • インターバル中はプレイヤーの移動や操作が可能

通貨の引き継ぎ

  • チーム通貨はマッチ全体で共有
  • ゲーム間で通貨が引き継がれる
  • 各ゲーム終了時に50Gのボーナス
  • 戦略的な通貨管理が重要

プレイヤーの扱い

  • マッチ開始時の参加者で固定
  • 途中参加・離脱は不可
  • 切断しても次のゲームで自動復帰
  • チーム構成は維持される

マッチ進行状況の表示

BossBar表示

先取モード

[先取3] 第2ゲーム | 赤1勝 青0勝 | 戦闘フェーズ - 8:45

固定ラウンドモード

[3/5] | 赤2勝 青1勝 | 建築フェーズ - 4:20

スコアボード表示

マッチ情報がスコアボードに追加表示されます：

  • 現在のゲーム数
  • 各チームの勝利数
  • マッチモード情報

戦略ガイド

通貨管理

  • 序盤温存: 最初のゲームで通貨を貯める
  • 中盤投資: 重要なゲームで大量購入
  • 終盤決戦: 最終ゲームに全力投入

チーム戦術

  • 役割固定: マッチ全体で同じ役割を継続
  • 学習と適応: 相手の戦術を分析して対策
  • モメンタム: 連勝の勢いを活かす

心理戦

  • プレッシャー: リードされても諦めない
  • 油断禁物: リードしていても気を抜かない
  • チーム士気: 声かけでモチベーション維持

マッチ終了

勝敗決定

  • 先取モード: 目標勝利数到達で即終了
  • 固定ラウンド: 全ゲーム終了後に判定

結果表示

=== マッチ終了 ===
赤チームがマッチに勝利！
最終スコア: 赤3勝 - 青1勝

設定可能な項目

デフォルト設定（config.yml）

match:
  default-mode: "first_to_x"
  default-target: 3
  interval-duration: 5

カスタマイズ可能な要素

  • マッチモード（先取/固定）
  • 目標値（勝利数/ラウンド数）
  • ゲーム間インターバル時間
